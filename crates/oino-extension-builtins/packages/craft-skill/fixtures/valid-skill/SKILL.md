# Release Notes

Create release notes that summarize user-visible changes, fixes, risks, and validation.

## Workflow

1. Inspect the provided commits, changelog, or milestone notes.
2. Group changes by user impact.
3. Write concise bullets with clear labels.
4. Call out breaking changes, migrations, and follow-up checks.

## Guardrails

- Do not invent shipped behavior.
- Ask for missing version or audience details when they affect wording.
