## Documentation rules

- Keep docs Oino-focused and useful to users, extension authors, or maintainers.
- Put public docs under this top-level `docs/` directory.
- Do not add hidden workflow files, implementation scratch notes, or agent planning logs to public docs.
- Avoid documenting compatibility with external agent APIs as a product promise; document Oino-native behavior instead.
